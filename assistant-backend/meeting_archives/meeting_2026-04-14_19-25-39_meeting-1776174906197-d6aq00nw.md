### 1. The Core Strategy
I built Shadow OS as a context-aware "Second Brain" designed to bridge professional productivity and personal well-being. The core strategy was to create a "bicameral" AI system—split into Shadow and Zenith personas—that could handle everything from objective work tracking to empathetic life coaching. To make this work, I needed a backend that could handle asynchronous AI orchestration while maintaining strict data privacy, which is why I chose a stack centered around FastAPI, LangGraph, and a hybrid database approach.

### 2. Technical Execution
On the frontend, I used React with Vite. For security, I implemented a zero-knowledge architecture using the Web Crypto API; sensitive notes are encrypted in the browser via AES-GCM 256-bit before they ever hit the network. 

The backend is a FastAPI server that manages authentication via Argon2 and JWT. The "brain" of the system is powered by LangGraph, which acts as the orchestrator. When a user sends a request, LangGraph determines if it needs a standard conversational chain or a RAG-augmented chain. For the latter, it performs a similarity search against a Pinecone vector database using Google's embedding-001 model, then feeds that context into Google Gemini 1.5 Flash for the final response. I also integrated the Google Calendar API via OAuth2 to allow the AI to handle natural language scheduling with two-way synchronization.

### 3. The Outcome
The biggest technical win was solving "context pollution." I engineered a "Memory Wall" using a PydanticOutputParser to classify notes as Activity, Idea, or Rant. I then wrote a dynamic filter that programmatically drops "Rant" documents when the user is in productivity mode, ensuring 100% context isolation. 

I also solved a critical multi-tenant security risk in Pinecone by injecting a hard metadata filter—specifically the `user_id`—directly into every query payload. This ensures that one user can never mathematically retrieve another user's memory fragments. The entire system is containerized with Docker, allowing the FastAPI backend to remain stateless and horizontally scalable.
