tell me about Advanced RAG Pipeline project

### assistant
I built an Advanced RAG Pipeline that solves the critical problem of LLM hallucinations and poor retrieval accuracy when querying complex academic documents like research papers and course syllabi. The system uses a two-stage hybrid retrieval architecture where I combined FAISS for semantic search with BM25 for exact keyword matching, then passed the top candidates through a HuggingFace Cross-Encoder reranker to filter out noise before feeding context to Google Gemini. I had to engineer a custom document ingestion pipeline using UnstructuredIO because standard PDF loaders were completely scrambling multi-column layouts and destroying table formatting in academic papers. The hybrid search design was essential because pure vector search would fail on exact queries like model names or dates, while pure keyword search missed conceptual questions entirely. I configured the final LLM stage at zero temperature with strict anti-hallucination prompts to guarantee factual accuracy. The result was a production-ready RAG system with near-zero hallucination rates that could handle both broad conceptual queries and precise entity extraction across thousands of institutional documents.
