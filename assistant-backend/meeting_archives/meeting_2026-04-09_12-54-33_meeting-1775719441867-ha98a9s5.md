I built an Advanced RAG Pipeline that combines hybrid search with cross-encoder reranking to create a highly accurate question-answering system for complex academic documents. The tech stack includes Python, LangChain, Google Gemini 1.5 Flash, FAISS for dense retrieval, BM25 for sparse keyword matching, and HuggingFace models like all-MiniLM for embeddings and BGE-Reranker for reranking. The primary technical challenge I solved was eliminating LLM hallucinations and bridging the gap between semantic understanding and exact keyword matching, which standard vector search completely fails at. I engineered a two-stage retrieval system where FAISS and BM25 first fetch the top 10 candidates in a 70-30 weighted ensemble, then a computationally heavy Cross-Encoder reranks them and filters down to only the top 3 highest-quality chunks before feeding them to the LLM. I also tackled complex PDF parsing issues by using UnstructuredIO to preserve multi-column layouts and table structures that standard loaders would scramble. The result was near-zero hallucination rates and a system that handles both conceptual questions and exact entity queries with production-grade accuracy.
