### assistant
Cursor (or Gursa) uses a transformer‑based LLM that predicts the next token given the code context. The model was pre‑trained on massive public codebases, so it learns common APIs and idioms. During inference it scores all possible tokens and picks the highest‑probability one, optionally filtered by a safety layer. To reduce hallucinations it conditions on the exact surrounding syntax and can invoke a retrieval module for relevant snippets. The retrieval step queries a vector DB of indexed project files, injecting those results as additional context. A low temperature and top‑k sampling further keep outputs deterministic and grounded in the retrieved code. If the model’s confidence drops, the system falls back to a human‑in‑the‑loop suggestion or blocks the completion.
