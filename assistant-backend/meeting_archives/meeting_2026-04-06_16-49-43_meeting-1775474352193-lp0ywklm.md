### 1. Optimal Approach  
Since we have a list of ~1 M strings, each 10‑30 chars, and we need to compute metrics (e.g., total length, longest string, frequency of a particular substring), we can do a single pass.  
- Iterate once, accumulating totals.  
- Use a dictionary for frequency counts if needed.  
- Avoid any extra data structures that grow with input size beyond the dictionary for frequencies.  

This gives O(N) time and O(K) space, where K is the number of distinct items we’re counting (e.g., distinct substrings).  

### 2. Time & Space Complexity  
- **Time:** O(N) – one pass over the array.  
- **Space:** O(K) – for the frequency dictionary; otherwise O(1).

def analyze_strings(strings):
    """
    Analyze an array of strings for:
      - total number of strings
      - total combined length
      - longest string
      - frequency of each string (optional)
    Parameters:
        strings (list[str]): List of input strings.
    Returns:
        dict: Dictionary containing the metrics.
    """
    # Initialize metrics
    total_strings = 0
    total_length = 0
    longest = ""
    freq = {}  # optional frequency count

    # Single pass over the list
    for s in strings:
        total_strings += 1
        total_length += len(s)

        # Track longest string
        if len(s) > len(longest):
            longest = s

        # Frequency count
        freq[s] = freq.get(s, 0) + 1

    # Build result
    result = {
        "total_strings": total_strings,
        "total_length": total_length,
        "longest_string": longest,
        "frequency": freq,  # remove if not needed
    }
    return result

# Example Usage

### 1. Optimal Approach  
Since we need to check a single substring against up to 1 M strings, the fastest thing is a single linear scan: iterate over the list once, use Python’s built‑in `in` operator to test membership, and record the index if found.  
The `in` operator on a string is a fast C implementation that stops as soon as the pattern is found, so it’s O(len(string)) in the worst case but usually much faster.  
We’ll build a result list of tuples `(index, found_index)` where `found_index` is the position of the substring or `-1` if absent. That keeps memory usage minimal—just a list of integers for 1 M entries.

### 2. Detailed Complexity Analysis  
**Time**:  
- We loop over each of the 1 M strings once: that’s 1 M iterations.  
- For each string, the `in` check scans at most the string’s length (10–30 chars). In the worst case, if the substring isn’t present, it scans the whole string. So per string the cost is O(L) where L ≤ 30.  
- Total time ≈ Σ O(L_i) over all strings. With L_i ≤ 30, the worst‑case total is 30 M character comparisons, which is trivial for modern CPUs.  
- The overhead of appending to the result list is O(1) per iteration, so it doesn’t change the asymptotic bound.  
**Space**:  
- We store the result list of length 1 M, each entry a small tuple of two integers. That’s roughly 16 bytes per entry → ~16 MB total.  
- No additional large structures are created; the input list itself is reused.  
- Python’s string immutability means each string is stored once; we never copy them.  
Thus the algorithm scales linearly in time and uses a modest, predictable amount of memory.

def find_substring_in_list(strings, pattern):
    """
    Scan a list of strings and return a list of tuples:
        (index_of_string, index_of_pattern_or_-1)
