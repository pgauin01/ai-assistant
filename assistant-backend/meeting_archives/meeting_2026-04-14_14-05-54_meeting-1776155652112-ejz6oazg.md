**The Question:** What's the difference between bind, call, and apply in JavaScript?

* All three methods let you explicitly set the `this` context when invoking a function.
* **call** invokes the function immediately with arguments passed individually: `func.call(thisArg, arg1, arg2)`.
* **apply** also invokes immediately but takes arguments as an array: `func.apply(thisArg, [arg1, arg2])`.
* **bind** doesn't invoke immediately—it returns a new function with `this` permanently bound: `const boundFunc = func.bind(thisArg)`.

### 3. Technical Deep Dive
**All three methods control the `this` context**, but they differ in invocation timing and argument handling. **`call` executes the function immediately** and accepts arguments as a comma-separated list (`func.call(context, arg1, arg2)`). **`apply` also executes immediately** but takes arguments as an array (`func.apply(context, [arg1, arg2])`), making it ideal when you have a dynamic argument list. **`bind` is fundamentally different**—it doesn't invoke the function at all. Instead, it returns a new function with the `this` context permanently locked in, which you can call later (`const boundFunc = func.bind(context); boundFunc(arg1, arg2)`).

**The classic use case for `bind`** is fixing the `this` context in React class components when passing event handlers (`this.handleClick = this.handleClick.bind(this)` in the constructor). **`call` and `apply` shine in functional programming patterns** like function borrowing—for example, using `Array.prototype.slice.call(arguments)` to convert an array-like object into a real array before ES6 spread syntax existed. **The mnemonic I use is "A for Array"**—`apply` takes an **A**rray of arguments, while `call` takes a **C**omma-separated list.
