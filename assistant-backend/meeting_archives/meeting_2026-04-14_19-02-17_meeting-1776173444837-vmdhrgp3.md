### 2. Technical Execution

I built the backend on **FastAPI** with full async support to handle slow LLM inference without blocking. The orchestration layer uses **LangGraph** as a state machine that routes requests between standard conversational chains and RAG-augmented chains. For memory, I implemented a dual-database architecture: **MongoDB** handles structured data like events and user profiles, while **Pinecone** stores vectorized embeddings of all user notes and activities using Google's embedding model.

The critical engineering piece was the "Memory Wall." I built a stream processor using `PydanticOutputParser` that forces the LLM to classify every incoming note as `ACTIVITY`, `IDEA`, or `RANT`. Then I wrote a dynamic filter function that intercepts Pinecone retrieval results. When Shadow mode is active, the backend programmatically drops any document tagged as a Rant before it reaches **Google Gemini 1.5 Flash** for inference. This guarantees 100% context isolation.

For privacy, I implemented zero-knowledge encryption using the browser's **Web Crypto API**. Personal journal entries are encrypted client-side with AES-GCM 256-bit before ever hitting the network. The FastAPI backend only stores base64 ciphertext.

### 3. The Outcome

The result is a production-ready system that's fully containerized with **Docker** and features two-way **Google Calendar sync** for natural language scheduling. The Memory Wall achieved perfect persona isolation, and the zero-knowledge architecture means even a full database breach wouldn't expose plaintext notes.
