tell me about Advanced RAG Pipeline project

### assistant
I built an Advanced RAG Pipeline that solves the critical problem of LLM hallucinations and poor retrieval accuracy when querying complex academic documents like research papers and course syllabi. The system uses a two-stage hybrid retrieval architecture where I combined FAISS for semantic search with BM25 for exact keyword matching, then passed the top candidates through a HuggingFace Cross-Encoder reranker to aggressively filter out noise before feeding only the highest-quality context to Google Gemini 1.5 Flash. One of the biggest technical challenges was parsing multi-column PDFs and complex document layouts, which I solved by integrating the Unstructured library with advanced layout detection to preserve the logical reading order and semantic structure of academic content. I also had to balance the computational cost of the Cross-Encoder by strategically limiting it to only rerank the top 10 hybrid results rather than the entire corpus, which gave me the precision of a heavy model while maintaining sub-second response times. The final system achieved near-zero hallucination rates by operating Gemini at temperature zero with a strict anti-hallucination prompt that forces it to declare unknown when context is missing. This architecture is production-ready for enterprise deployments where factual accuracy is absolutely mission-critical.

### user
Quick Answer

### assistant
MCP is Model Context Protocol, an open standard created by Anthropic that acts like USB-C for AI systems. It lets LLMs securely connect to external tools, databases, and APIs without writing custom integration code for every single data source. The protocol standardizes how context flows between the model and external systems through a client-server architecture where MCP servers expose resources like file systems or databases and MCP clients like Claude Desktop consume them. It's critical because it solves the nightmare of maintaining dozens of fragile custom connectors when you need your agent to access Slack, GitHub, your internal docs, and production databases all in one workflow. Instead of hardcoding API calls into your agent logic
