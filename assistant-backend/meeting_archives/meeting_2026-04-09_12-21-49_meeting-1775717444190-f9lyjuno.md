tell me about Advanced RAG Pipeline project

### assistant
I built an Advanced RAG Pipeline that combines hybrid search with reranking to solve the critical problem of LLM hallucinations when querying complex academic documents. The system uses Python, LangChain, and Google Gemini 1.5 Flash as the LLM, with a two-stage retrieval architecture where FAISS handles dense semantic search and BM25 handles exact keyword matching. The biggest technical challenge was that pure vector search would completely fail on specific queries like model names or dates, while also passing too much noisy context to the LLM causing hallucinations. I solved this by implementing a hybrid ensemble retriever that merges results from both engines, then passes the top candidates through a HuggingFace Cross-Encoder reranker that aggressively filters down to only the three most relevant chunks. I also had to engineer a custom document ingestion pipeline using UnstructuredIO because standard PDF loaders were scrambling multi-column research papers and destroying table formatting. The final result was a near-zero hallucination rate system that can handle both conceptual questions and exact entity extraction with sub-second response times, making it viable for enterprise deployment.

The RAG pipeline prevents hallucinations by never letting the LLM guess. It forces the model to only answer from the exact chunks retrieved from the vector database. The hybrid search is critical because FAISS finds semantically similar content while BM25 catches exact keyword matches the model would otherwise miss. The reranker then scores all candidates and only passes the top three most relevant chunks to Gemini, so the LLM sees minimal noise. I also engineered the prompt template to explicitly instruct the model to say "I don't know" if the answer isn't in the provided context. This constraint-based approach is why it achieves near-zero hallucination rates compared to a raw LLM call.
