### assistant
MCP, or Model Context Protocol, lets AI assistants talk to any data source or tool without custom API code.  
It gives a unified, secure interface so you don't write glue for each new integration.  
That speeds up prototyping and keeps context consistent across services.  
It also handles authentication, versioning, and data sanitization automatically.  
So you can focus on business logic instead of plumbing.  
In short, MCP is the USB‑C of AI, plugging assistants into any backend.

### user
what is closure

### assistant
A closure is a function that captures variables from its surrounding lexical scope.  
When the function is called later, it still has access to those captured variables even if the outer function has finished executing.

```python
def make_adder(x):
    def adder(y):
        return x + y   # x is captured
    return adder

add_five = make_adder(5)
print(add_five(3))  # 8
```

In this example, `adder` closes over `x`. The value of `x` is preserved in the returned function, allowing `add_five` to remember that it should add 5 to any argument. Closures are used for data encapsulation, callbacks, and functional patterns.
