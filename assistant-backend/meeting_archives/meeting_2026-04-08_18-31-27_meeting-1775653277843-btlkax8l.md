### 1. The Elevator Pitch  
Cursor’s autocomplete works by feeding a Fill‑In‑The‑Middle (FIM) prompt that combines precise AST context, open‑file tokens, and a tiny project‑wide retrieval slice into a LLM, then validates the suggestion with a fast “shadow compilation” step.

### 2. Core Mechanics

The AST, local tokens, and retrieved snippets are concatenated into a FIM prompt; the model runs at low temperature with top‑p filtering, scoring continuations. ### 1. The Elevator Pitch  
MCP (Model Context Protocol) is an open, language‑agnostic schema that packages structured context—retrieval results, tool outputs, or agent state—into a validated payload the LLM consumes as a single, token‑budgeted block.

### 2. Core Mechanics  
Instead of splicing raw JSON or free‑form text, the producer serializes context to the MCP format, includes a token‑budget field, and signs it against a Pydantic schema. The LLM’s runtime parses the payload, extracts only the fields it needs, and discards anything exceeding the budget. This prevents prompt overflow, cuts TTFT latency, and eliminates hallucinations caused by missing or malformed context.

### 3. Top Options & Trade‑offs  
- **Naïve concatenation:** simplest but wastes tokens on delimiters, blows the context window, and forces the model to guess missing structure → higher cost and lower precision.  
- **MCP‑validated payload:** adds a tiny serialization step but guarantees token‑budget adherence and strict type safety → better recall, lower latency, and deterministic failures.  
- **Hybrid (MCP for agents, plain for single‑step RAG):** use full protocol when multiple tools interact; fall back to plain text when the pipeline is trivial.  
We start with MCP for any multi‑agent or tool‑rich flow; switch to plain only if latency budget is sub‑10 ms and context is under 256 tokens.

### 4. Production Example  
A LangGraph agent calls a vector store, receives top‑k documents, wraps them in an MCP payload with a 2 k token budget, and sends it to Claude. Claude parses the payload, sees the budget flag, and only attends to the top three snippets, returning a concise answer in 120 ms. Without MCP, the same call would exceed the model’s context, trigger truncation, and produce a hallucinated citation.
