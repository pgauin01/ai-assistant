### 1. The Elevator Pitch  
I’ve spent the last six years building and shipping microservices with a focus on speed and security. I run CI/CD in GitHub Actions, automate infra with Terraform and Helm on Kubernetes, and lock down everything with IAM, secrets, and automated scans.

### 2. Core Mechanics  
When a developer pushes code, GitHub Actions kicks off a pipeline: lint, unit test, build a Docker image, push to Azure Container Registry, then Helm deploys to a Kubernetes cluster. Terraform provisions the cluster, VPC, and IAM roles. Secrets live in Azure Key Vault and are injected at runtime via managed identities, so no credentials ever hit the repo. Prometheus and Grafana surface metrics; alerts go to Slack. For data protection, I enforce bucket policies and VPC endpoints so traffic never leaves the private network.

### 3. Top Options & Trade‑offs  
- **GitHub Actions vs Jenkins vs GitLab CI**: GitHub Actions is tightly integrated with the repo, no extra server maintenance, and scales automatically. Jenkins gives you full control but you have to run and patch the master, which eats dev bandwidth. GitLab CI is great if you already run GitLab, but the UI feels clunky for large pipelines.  
- **Terraform + Helm vs Cloud‑Native Managed Services**: Terraform gives you declarative infra that’s versioned and portable. Helm lets you package and version your deployments. Managed services like Azure App Service or AWS Fargate reduce ops overhead but lock you into a vendor’s pricing model and can be slower to roll out custom networking.  
- **Secrets Management**: Azure Key Vault or AWS Secrets Manager are the gold standard—managed, audited, and integrate with IAM. Storing secrets in environment variables or config files is quick to set up but risks accidental leaks and is hard to rotate.

### 4. Production Example  
We had a new feature that required a database schema change. I added a migration job to the pipeline that runs against a staging database first. Once the migration passes, the same job deploys the updated container to a blue‑green Kubernetes deployment. The Helm chart pulls the new image tag, and Terraform updates the service mesh to route 10% of traffic to the new pod. Prometheus alerts if the latency spikes, and if everything looks good, we shift the remaining 90% traffic. All secrets are pulled from Key Vault at runtime, so the migration script never touches plaintext credentials. This process takes under an hour from commit to production, with zero manual intervention.
