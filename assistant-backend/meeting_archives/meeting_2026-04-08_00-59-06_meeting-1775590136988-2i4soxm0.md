### 1. Document Ingestion Stage
The Advanced RAG Pipeline project starts with a document ingestion stage where I utilized UnstructuredIO and PyPDF to parse complex academic formats and multi-column PDFs.
* **What happens:** The system extracts clean and semantically coherent text from the documents, which is then used for indexing and querying.
* **Why this matters (Trade-offs):** This step is crucial for ensuring that the system can accurately retrieve relevant information, and using UnstructuredIO and PyPDF allows for efficient parsing of complex document layouts.
* **Input & Output:** The input is a set of PDF documents, and the output is a collection of parsed text documents ready for indexing.

### 2. Indexing Stage
The parsed text documents are then indexed using a combination of FAISS for dense retrieval and BM25 for sparse retrieval.
* **What happens:** The system creates a dense index using FAISS and a sparse index using BM25, allowing for efficient querying and retrieval of relevant documents.
* **Why this matters (Trade-offs):** Using both FAISS and BM25 provides a robust retrieval system that can handle both semantic and keyword-based queries.
* **Input & Output:** The input is the parsed text documents, and the output is a set of indexed documents ready for querying.

### 3. Query Stage
When a query is received, the system uses the indexed documents to retrieve relevant results.
* **What happens:** The system uses the dense and sparse indexes to retrieve a set of relevant documents, which are then ranked and filtered using a Cross-Encoder Reranker.
* **Why this matters (Trade-offs):** The Cross-Encoder Reranker helps to reduce LLM hallucinations and ensure factual accuracy, providing more reliable results.
* **Input & Output:** The input is a user query, and the output is a set of relevant documents with their corresponding rankings and scores.

### 4. Reranking Stage
The retrieved documents are then reranked using a Cross-Encoder Reranker to ensure factual accuracy and reduce LLM hallucinations.
* **What happens:** The system uses the Cross-Encoder Reranker to re-evaluate the relevance of the retrieved documents and adjust their rankings accordingly.
* **Why this matters (Trade-offs):** This step helps to improve the accuracy and reliability of the results, providing a better user experience.
* **Input & Output:** The input is the set of retrieved documents, and the output is a revised set of documents with updated rankings and scores.
